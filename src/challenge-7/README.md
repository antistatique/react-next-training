# Challenge #7 - Next.js Router (~6h)

## 🔢 Objectifs

1. Comprendre le routing de Next.js
2. Pouvoir utiliser des liens, paramètres, etc.

## 📋 Consignes

Dans la nouvelle branche dédiée à Next.js, nous allons créer une plateforme dédiée aux Pokemons. Pour commencer, il s'agit de créer des pages pour :
- Lister et paginer tous les Pokemons
- Afficher le détail de chaque Pokémon
- Afficher les informations de la plateforme (contenu simple et libre)
- Afficher les informations de contact (contenu simple et *fake* autorisé)

Pour naviguer entre les pages, il faudra utiliser le composant `<Link />` comme indiqué dans la documentation officielle. Le “moment” de récupération des données (fonctions de data-fetching de Next.js ou hooks) est laissé libre pour l'instant. Il faut également ajouter un *header* pour pouvoir naviguer entre les pages principales. Pour finir, on doit pouvoir être capable d'accéder directement à une page du listing via l'URL.

Le design se base sur les composants de TailwindUI et quelques améliorations peuvent être entreprises: couleurs et police de caractère.

## 🗜️ Contraintes

- N'utiliser que les documentations officielles pour avancer sur le projet
- Ne pas copier/coller de code d'autres projets existants
- Pas d'erreur de *lint*
